# Release notes (更新内容)

更新完成后，新版本启动时会弹一个「更新内容」窗口，显示**本版本**的 notes，按用户的语言偏好（`config.json` 的 `lang`，中/英）选择。

## 约定
- 每次发版（bump 根目录 `VERSION` 到 `X.Y.Z`）必须同时新增两份文件：
  - `notes/X.Y.Z.zh.md` — 中文
  - `notes/X.Y.Z.en.md` — English
- 内容是面向**用户**的简短「这次更新了什么」（不是内部实现细节）。纯文本/简单 Markdown 即可。
- 缺失时优雅降级：找不到当前语言 → 退到英文 → 再退到中文 → 都没有则不弹窗、仅保留原来的「已更新到 vX」通知。

## 为什么「下次更新」就能看到（不是下下次）
弹窗由「重启后的**新**版本」在启动时画（读 `update_result.txt` 面包屑得知刚更新到 vX，再读 `notes/vX.<lang>.md`）。新版本正好就是加了这功能、且自带自己 notes 的版本，所以加了功能的那次发布、用户更新上来时即可见。

## 不弹窗的情形（设计如此）
- 静默的 Python→Rust 迁移（按 percent 灰度、用户没主动点更新）：不写 `update_result.txt` 面包屑 → 不弹窗。仅用户**主动**版本更新才弹。
